Freeablo s source code is split up into two apps and components Apps can depend on and components can depend on but components cannot depend on apps The idea of this arrangement is to separate out common code that can be used in multiple programs into common so for example the launcher and the main game executable can both use the mpq file IO library Tests live in the tests folder See ref md_docs_tests I won t try to list all the components as it s sure to go out of but I ll gloss over a few of the most important ones responsible for loading Diablo s CEL and CL2 image files into plain rgba buffers There is some documentation available on these files here
